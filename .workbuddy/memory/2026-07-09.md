# 2026-07-09 工作日志

## MySQL 勒索攻击修复（严重安全事件）

### 事件经过
- 用户报告服务无法显示，排查发现后端日志报 `Unknown database 'pet_feeding'`
- MySQL 数据库被攻击者删除，替换为 `RECOVER_YOUR_DATA` 勒索数据库
- 勒索信息要求支付 0.0133 比特币
- 根因：MySQL 3306 端口暴露公网（0.0.0.0:3306）+ root 密码极弱（`123456`）

### 修复措施
1. **docker-compose.yml 安全加固**：
   - MySQL 端口从 `3306:3306` 改为 `127.0.0.1:3306:3306`（仅本地访问）
   - root 密码从 `123456` 改为 `PetFeed_DB_R00t_2026!`
   - 后端 SPRING_PROFILES_ACTIVE 从 `dev` 改为 `prod`
2. **删除被污染的 MySQL 数据卷**，重新初始化
3. **重建 pet_feeding 数据库**：执行 init_prod_db.sql（7张表+种子数据），创建 petfeeder 用户
4. **修复 application-prod.yml**：
   - 数据库 URL 从硬编码 `101.42.24.114:3306` 改为环境变量 `${MYSQL_HOST:mysql}:${MYSQL_PORT:3306}`
   - JWT 过期时间从 `86400000`（约1000天）修正为 `86400`（24小时）
5. **宠物 species 字段修正**：种子数据从中文（'猫','狗'）改为枚举值（'CAT','DOG'），使前端 emoji 图案正常显示

### 验证结果
- 4个容器全部正常运行，MySQL 仅监听 127.0.0.1
- 登录 API `/api/user/login` 返回 200 + JWT Token
- 宠物列表 API 返回3条数据，中文正确，species 为 CAT/DOG
- Dashboard API 返回完整统计数据
